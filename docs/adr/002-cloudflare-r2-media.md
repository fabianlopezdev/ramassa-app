# 002. Cloudflare R2 for media storage

**Status:** Accepted
**Date:** 2026-04-09

## Context

The app needs media storage for forum images, profile photos, gallery items, and documents. Storage costs scale with reads (egress), not just writes. Players will view shared media frequently.

## Decision

Use Cloudflare R2 for all media storage. Upload via presigned URLs generated by a Cloudflare Worker (not Supabase Edge Function) because R2 binding is native in Workers.

## Alternatives Considered

- **Supabase Storage** — rejected. 1GB free tier, $0.021/GB/month, plus egress charges. Egress costs would grow with user activity.
- **AWS S3** — rejected. More expensive, adds another cloud provider to manage.

## Consequences

- 10GB free tier, zero egress fees, $0.015/GB/month beyond free tier
- Presigned URL Worker validates auth (Supabase JWT) and file constraints
- R2 CORS policy must be configured for PUT from app origins
- Same Cloudflare network as hosting — fast delivery
