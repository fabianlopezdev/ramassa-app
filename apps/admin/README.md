# Ramassa Admin (`apps/admin`)

TanStack Start (Vite, TanStack Router file routes) app for the staff CMS and the entity
portal (ADR-001 split, ADR-016 framework). Desktop-optimized, login-gated, closed access.

## Commands

```
bun run dev        # Dev server on http://localhost:3000
bun run build      # Production build (also regenerates src/routeTree.gen.ts)
bun run typecheck  # tsc --noEmit
bun test           # Tests (run from repo root so the happy-dom preload applies)
```

## Route skeleton

TanStack Router file routes under `src/routes/`:

- `login.tsx` — `/login`
- `_staff.tsx` + `_staff.dashboard.tsx` — pathless staff layout, `/dashboard`
- `_entity.tsx` + `_entity.portal.tsx` — pathless entity layout, `/portal`
- `index.tsx` — `/` redirects to `/login`

`src/routeTree.gen.ts` is generated by the TanStack Router Vite plugin (dev/build). It is
committed but excluded from eslint and prettier.

## Deployment target: Cloudflare Workers (hard constraint)

This app deploys to **Cloudflare Workers** via `@cloudflare/vite-plugin` (ADR-016); the
deploy wiring lands in RAPP-15 (`wrangler.jsonc` with `compatibility_flags:
["nodejs_compat"]`, `main: "@tanstack/react-start/server-entry"`).

Consequence from day one: **server functions and any server-side code run on Workers, not
Node**. Before using any Node API (`node:fs`, `node:crypto`, etc.) in a server function,
check it is covered by `nodejs_compat`:
https://developers.cloudflare.com/workers/runtime-apis/nodejs/

## Styling

Tailwind CSS v4 (`@tailwindcss/vite`) + shadcn/ui (radix-nova preset, RTL mode: logical
`ps-`/`pe-` properties per the repo RTL rule). Theme variables live in
`src/styles/app.css`; they are a hook point until the shared design tokens package
(RAPP-9) feeds them.

## TanStack Start work is official-docs-first (repo hard rule 13)

Consult https://tanstack.com/start/latest and https://tanstack.com/router/latest before
writing any Start/Router code and verify APIs against the installed version. Known local
deviation: the docs' `resolve.tsconfigPaths: true` vite option needs a newer Vite than the
installed 7.x, so `vite.config.ts` mirrors the `@/*` tsconfig path with an explicit alias.
